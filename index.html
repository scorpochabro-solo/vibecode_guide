Как это вообще работает

AI-агент — ваш сотрудник

Программа, которая сама пишет код: создаёт файлы, исправляет ошибки, показывает результат. Вы ставите задачу и принимаете работу — как руководитель.

Промпт — техзадание словами

Обычный текст, которым вы объясняете, что хотите получить. Никакого программирования — только внятное описание задачи.

Качество промпта = качество продукта

Агент делает ровно то, о чём его попросили. Конкретный промпт даёт конкретный продукт, расплывчатый — «в среднем по интернету». Этому гайд и учит.

01

Регистрация и приложение

Проверено: 11 июля 2026

Заводим аккаунт, оформляем план и ставим приложение Claude на Windows. Здесь ещё ничего не может пойти не так.

  1. 1

    Создайте аккаунт на claude.ai

    Откройте claude.ai и нажмите «Continue with Google» (вход через Google-почту) или «Continue with email». Пароля у аккаунта Claude нет вообще: при входе по почте вам приходит письмо со ссылкой — нажали и вошли.

  2. 2

    Оформите план Pro или выше

    Claude Code — агент, ради которого мы здесь, — входит в планы Pro, Max, Team и Enterprise. В бесплатном плане его нет. Актуальные цены смотрите на официальной странице тарифов — они меняются, поэтому здесь мы их не пишем.

    Не спешите платить прямо сейчас: главы 1.2 и 1.3 (первые промпты и «Генератор промптов») можно пройти и на бесплатном плане. Подписка понадобится с главы 1.4.

  3. 3

    Установите приложение Claude для Windows

    Зайдите на claude.com/download и нажмите «Download for Windows» (для обычного компьютера — версия x64). Откройте скачанный файл, дождитесь установки и запустите Claude из меню «Пуск». Войдите в тот же аккаунт. Требование — Windows 10 или новее.

  4. 4

    Осмотритесь в приложении

    Слева три главные зоны: Chat — обычные чаты с Claude; Projects — «папки»-проекты, у каждого свои инструкции (пригодится в главе 1.3); Code — вкладка, где живёт Claude Code, наш агент-программист.

    Вкладка Code попросит Git

    Для работы вкладки Code нужна бесплатная программа Git for Windows. Приложение само подскажет; поставьте её (при установке отметьте «Add to PATH») и перезапустите Claude. Что такое Git — в глоссарии, а пока это просто «зависимость, которую попросили».

Интерфейс Claude обновляется часто. Если кнопка называется чуть иначе или переехала — это нормально, ориентируйтесь по смыслу.

02

Первые промпты: 15-минутный практикум

Промпт — это техзадание словами. Здесь мы научимся отличать промпт, который даёт результат, от промпта, который даёт «что-то среднее».

Откройте обычный чат (вкладка Chat) и попробуйте оба варианта ниже. Разница в ответах убедит лучше любых объяснений.

Плохой промпт

«Сделай мне красивый современный сайт про мою кофейню.»

Хороший промпт

«Я открываю кофейню «Утро» в жилом районе. Аудитория — жители соседних домов 25–45 лет, которые берут кофе по пути на работу. Задача: придумай структуру одностраничного сайта. Критерии: тёплая светлая гамма, максимум 5 секций, обязательно карта и часы работы, никаких стоковых фраз вроде «уникальная атмосфера». Формат ответа: список секций, и для каждой — заголовок и 1–2 предложения текста.»

На «плохой» промпт вы получите усреднённый шаблон: агент не телепат и заполнит пробелы догадками. «Хороший» промпт отличается не длиной, а тем, что в нём есть четыре части.

Правило четырёх частей

Контекст (кто вы и что у вас) → Задача (что именно сделать) → Критерии (что обязательно, что запрещено) → Формат ответа (в каком виде хотите результат). Проверяйте каждый свой промпт по этим четырём пунктам — этого достаточно, чтобы обгонять 90% пользователей.

Три упражнения (по 5 минут)

  1. 1

    Дорастите промпт

    Возьмите фразу «нужен сайт-визитка для фотографа» и перепишите её по правилу четырёх частей. Отправьте оба варианта в чат и сравните ответы.

  2. 2

    Добавьте запреты

    К любому запросу допишите строку «Запрещено: …» — например, «канцелярит, восклицательные знаки, слово „уникальный“». Заметьте, насколько чище становится текст.

  3. 3

    Задайте формат

    Попросите один и тот же результат в трёх форматах: таблицей, списком шагов и коротким абзацем. Так вы поймёте, что формат ответа — тоже ваш инструмент.

Качество промпта = качество продукта. Всё остальное в этом гайде — способы сделать так, чтобы хорошие промпты писались за вас. Начнём в следующей главе.

03

Проект «Генератор промптов»

Проверено: 11 июля 2026

Собираем себе персонального промпт-инженера: кидаете в проект сырую идею — получаете профессиональное ТЗ для агента.

  1. 1

    Создайте проект

    В приложении Claude откройте Projects → «New project». Назовите его «Генератор промптов».

  2. 2

    Откройте инструкции проекта

    Внутри проекта найдите поле инструкций (Project instructions / «Инструкции»). Это «должностная инструкция»: Claude будет следовать ей в каждом чате этого проекта.

  3. 3

    Вставьте системный промпт

    Скопируйте промпт ниже кнопкой и вставьте в инструкции проекта целиком. Сохраните.

  1. 4

    Проверьте на тестовой идее

    Напишите в чат проекта что-нибудь сырое: «хочу сайт для выгула собак». В ответ придёт не сайт и не поток воды, а до 5 уточняющих вопросов — а после ваших ответов готовый профессиональный промпт со структурой, дизайном и чеклистом готовности.

Как это работает: вы один раз вложили «мозги промпт-инженера» в инструкции проекта, и теперь любая идея, брошенная в этот проект, возвращается техзаданием, которое не стыдно отдать агенту-программисту. Этот промпт универсален — в Части 2 мы вставим его же в ChatGPT.

Бонус

Projects доступны даже на бесплатном плане (до 5 проектов) — генератор можно собрать ещё до оформления подписки.

04

Установка Claude Code на Windows

Проверено: 11 июля 2026

Самая «страшная» глава — с терминалом. На деле: одна команда, три минуты, и у вас на компьютере живёт агент-программист.

Микро-урок: что такое PowerShell и как его открыть

Терминал — это окно, где компьютером управляют текстовыми командами, а PowerShell — стандартный терминал Windows. Выглядит как чёрное окно из фильмов про хакеров, но бояться нечего: команды из этого гайда ничего не ломают, а незнакомую команду всегда можно просто не выполнять.

Как открыть: нажмите клавиши Win+X и в меню выберите «Терминал» (Windows 11) или «Windows PowerShell» (Windows 10). Признак, что вы в правильном месте: строка начинается с PS C:\….

Не открывайте «Windows PowerShell (x86)» из меню «Пуск» — из него установка выдаст ложную ошибку про 32-битную Windows даже на нормальном компьютере.

  1. 1

    Выполните команду установки

    Скопируйте команду, вставьте в PowerShell (правая кнопка мыши или Ctrl+V) и нажмите Enter. Права администратора не нужны, WSL не нужен. Требования: Windows 10 1809 или новее, 4 ГБ памяти (официальная инструкция).

    PowerShell
    > irm https://claude.ai/install.ps1 | iex
  2. 2

    Перезапустите терминал и проверьте

    Закройте окно PowerShell, откройте новое (это важно — так терминал «узнаёт» о новой программе) и выполните:

    PowerShell
    > claude --version

    Если в ответ пришёл номер версии — установка удалась.

  3. 3

    Создайте папку проекта

    Агенту нужна отдельная папка под каждый проект — его «рабочий стол». Команда mkdir создаёт папку, cd — заходит в неё:

    PowerShell
    > mkdir C:\projects\my-first-site; cd C:\projects\my-first-site
  4. 4

    Запустите Claude Code и войдите

    Наберите claude и нажмите Enter. При первом запуске агент предложит войти: откроется браузер — авторизуйтесь тем же аккаунтом, что и в главе 1.1. Так подписка Pro привяжется к агенту.

    PowerShell
    > claude

    Браузер не открылся? Нажмите клавишу «c» — ссылка для входа скопируется, вставьте её в браузер вручную.

Git for Windows — по желанию (но пригодится)

Для Claude Code в терминале Git не обязателен: без него агент выполняет команды через PowerShell. Но с ним агент получает более привычный инструмент Bash, а вкладке Code в приложении (глава 1.1) Git нужен обязательно. Ставьте с галочкой «Add to PATH» и перезапускайте терминал.

Если что-то пошло не так

Пишет «claude is not recognized…»

Сначала просто откройте новое окно терминала. Не помогло — выполните две команды ниже (они добавляют программу в «список известных» Windows) и снова откройте новый терминал.

PowerShell
> $currentPath = [Environment]::GetEnvironmentVariable('PATH', 'User')> [Environment]::SetEnvironmentVariable('PATH', "$currentPath;$env:USERPROFILE\.local\bin", 'User')

Что-то работает странно

У Claude Code есть встроенный «врач» — он сам проверит установку и скажет, что не так:

PowerShell
> claude doctor

Как показать агенту скриншот?

Сделайте скриншот (например, Win+Shift+S), затем в окне Claude Code нажмите Alt+V — именно Alt+V, а не Ctrl+V. В редких терминалах сочетание может отличаться.

Установщик ругается, что файл занят

Закройте другие окна установки, удалите папку загрузок установщика командой ниже и запустите установку заново.

PowerShell
> Remove-Item -Recurse -Force "$env:USERPROFILE\.claude\downloads"

Как обновлять и выходить?

Обновление: claude update (обычно агент обновляется сам). Выход из сессии: команда /exit. Продолжить вчерашний разговор в этой же папке: claude -c.

05

Плагины: прокачиваем агента

Проверено: 11 июля 2026

Плагин — это «отправить сотрудника на курсы»: одна команда, и агент получает новые навыки. Ставим дизайн-мозг и памятку.

Словарь на минуту. Скилл — конкретный навык: папка с файлом-инструкцией, который агент читает, когда навык нужен. Плагин — «коробка», в которой скиллы (и другие расширения) распространяются: добавили источник — установили — пользуетесь. Полная механика — в официальной документации, но для старта хватит двух команд из карточки ниже.

Правило безопасности

Официальная документация предупреждает: плагины могут исполнять код на вашем компьютере. Ставьте только те, чьему автору доверяете. Оба плагина этой главы открыты и проверяемы.

UI UX Pro Max

Дизайн-мозг агента

GitHub плагина

База дизайн-знаний, которую агент использует при вёрстке: 67 UI-стилей, 161 цветовая палитра, 57 пар шрифтов, 99 UX-правил и 25 типов графиков. Без него агент верстает «как получится», с ним — подбирает стиль, палитру и шрифты под вашу задачу.

Понадобится

  • Python 3 — бесплатный язык, нужен скриптам плагина. Проверка: python --version в PowerShell. Если нет — поставьте с python.org или командой winget install Python.Python.3.12 (это наша подсказка; README отправляет на python.org или пакетный менеджер).

Путь А — внутри Claude Code (проще)

Запустите claude в папке проекта и выполните две команды по очереди:

Claude Code
> /plugin marketplace add nextlevelbuilder/ui-ux-pro-max-skill> /plugin install ui-ux-pro-max@ui-ux-pro-max-skill

Путь Б — через PowerShell

Понадобится Node.js (если нет — winget install OpenJS.NodeJS.LTS). Установите утилиту и включите скилл в папке проекта:

PowerShell
> npm install -g ui-ux-pro-max-cli> cd C:\projects\my-first-site> uipro init --ai claude

Проверка, что работает

Попросите агента: «Построй лендинг для спа-салона». Если в плане появились конкретный стиль, палитра и пара шрифтов из базы — плагин работает.

Грабли

  • Пакет называется ui-ux-pro-max-cli, а команда после установки — uipro. Старый пакет uipro-cli не ставьте.
  • В README команды написаны как python3 … — на Windows пишите python вместо python3.

06

Конвейер «Идея → Продукт»

Проверено: 11 июля 2026

Центральная глава. Соединяем всё в один процесс, который вы будете повторять для каждой новой идеи.

  1. 01Идея
  2. 02Генератор промптов
  3. 03Готовый промпт
  4. 04Фаза 1: план
  5. 05Ваше «ок»
  6. 06Фаза 2: код
  7. 07Просмотр
  8. 08Итерации
  9. 09Публикация

Два подсвеченных узла — сердце конвейера: агент сначала показывает план, и только после вашего «ок» пишет код.

Ключевой элемент конвейера — двухфазность. Сначала агент показывает план без единой строчки кода, вы читаете его и говорите «ок» (или поправляете). Недопонимание ловится на бумаге за минуту — а не в готовом коде за час. Именно поэтому и генератор из главы 1.3, и шаблон ниже требуют от агента: «план → стоп → одобрение → код».

Мастер-шаблон

Когда идея простая и генератор промптов кажется избыточным — берите шаблон напрямую. Заполните три места в квадратных скобках и отправьте агенту:

Итерации: как просить правки

После Фазы 2 агент расскажет, как открыть результат в браузере. Смотрите и просите правки по одной: «сделай шапку ниже», дождитесь результата, посмотрите, потом следующая. Десять маленьких итераций дают лучший результат, чем одна просьба «исправь всё сразу».

Публикация

Готовый продукт лежит у вас на компьютере, а хочется ссылку, которую не стыдно отправить. Деплой (публикацию сайта в интернете) агент тоже сделает сам — на бесплатных сервисах GitHub Pages или Vercel. Готовый промпт «Публикация» лежит в библиотеке промптов: он попросит агента провести вас даже через создание аккаунтов.

07

Типичные ошибки новичка

Девять граблей, на которые наступают все. Прочитайте один раз — сэкономите несколько вечеров.

«Сделай красиво» — не задача, а лотерея. Агент заполнит пробелы догадками, и получится «в среднем по интернету». Лечится правилом четырёх частей из главы 1.2: контекст → задача → критерии → формат.

Просьба «сайт + блог + личный кабинет + оплата» одной строкой почти гарантирует кашу. Стройте слоями: каркас → одна фича → посмотрели → следующая. Маленькие шаги легче проверять и легче откатывать.

Соблазн «давай сразу код» велик, но план — это 60 секунд чтения, которые ловят недопонимание до того, как оно размазано по файлам. Всегда требуйте план и говорите «ок» осознанно.

Агент спрашивает — это хорошо, а не «я что-то не так написал». Отвечайте честно, даже если ответ «не знаю, реши сам». Молчание или «делай как хочешь» на важные вопросы возвращают вас к лотерее.

Запускать агента в «Загрузках» или прямо в C:\ — плохая идея: агент видит лишние файлы, а его изменения перемешиваются с вашими документами. Одна идея = одна папка в C:\projects.

Если не открывать сайт в браузере после каждой правки, ошибки копятся незаметно, и потом непонятно, какой шаг всё сломал. Правило: шаг → посмотрел → следующий шаг.

Агент спрашивает разрешения на действия — например, выполнить команду или удалить файл. Читайте, на что соглашаетесь: одобрять не глядя — всё равно что подписывать документы не читая. Особенно осторожно с удалением и перезаписью.

«А давай ещё поменяем нишу и цвета» посреди вёрстки формы — способ получить хаос. Дожмите текущий шаг до «работает», а потом меняйте направление отдельной задачей.

Цены, телефоны, отзывы, фотографии агент честно знать не может — и хороший промпт запрещает ему их выдумывать. Всё, что знает только заказчик, помечается [В КВАДРАТНЫХ СКОБКАХ] и заполняется вами.

Codex на Windows

Проверено: 11 июля 2026

Основной путь — прямо в приложении, без терминала. Для смелых — Codex CLI одной командой.

  1. 1

    Создайте папку и добавьте её как проект

    Создайте папку для проекта (можно обычным Проводником: правая кнопка → «Создать» → «Папку», например C:\projects\my-codex-site). В режиме Codex нажмите Add new project (или Ctrl+O) и выберите эту папку.

  2. 2

    Дайте первую задачу

    Напишите задачу в поле снизу (подойдёт мастер-шаблон из главы 1.6 или результат генератора). Агент работает прямо на вашем компьютере: команды выполняет в PowerShell, а защищает вас песочница — ограда, внутри которой агент свободно работает с файлами проекта, но за её пределы и в интернет ходит только с вашего разрешения (как это устроено).

    Держите под полем ввода режим «Ask for approval» — так песочница остаётся включённой, а агент спрашивает разрешение на рискованные действия.

  3. 3

    Поставьте инструменты, которые рекомендует OpenAI

    Официальная документация советует поставить три бесплатных инструмента — с ними агент работает заметно лучше (Git нужен панели проверки изменений, Node.js и Python — для сборки сайтов). Вставьте в PowerShell по одной строке:

    PowerShell
    > winget install --id Git.Git> winget install --id OpenJS.NodeJS.LTS> winget install --id Python.Python.3.14

    Из той же официальной страницы, для продвинутых: winget install --id Microsoft.DotNet.SDK.10 и winget install --id GitHub.cli (после него — gh auth login).

Для смелых: Codex CLI в терминале

Тот же агент существует в виде программы для терминала — как Claude Code. Это необязательный шаг: приложения выше достаточно. Установка одной командой из официального репозитория:

PowerShell
> powershell -ExecutionPolicy ByPass -c "irm https://chatgpt.com/codex/install.ps1 | iex"

Дальше знакомо: перезапустите терминал, зайдите в папку проекта (cd C:\projects\my-codex-site), запустите codex и выберите «Sign in with ChatGPT» — вход через браузер.

При первой настройке песочницы Windows покажет запрос администратора (UAC) — согласитесь, иначе настройка сорвётся. Это разовая операция самой Windows, а не «агент хочет власти».

07

Ошибки и отличия от Claude

Все девять граблей из главы 1.7 действуют и здесь. Ниже — то, что специфично для Codex.

Режим полного доступа снимает с агента ограничения: официальная документация прямо предупреждает, что в нём агент не ограничен папкой проекта и способен на разрушительные действия. Для задач этого гайда полный доступ не нужен: оставьте «Ask for approval».

Единственный UAC-запрос при первой настройке — это Windows создаёт песочницу. Нажать «Нет» — самый частый способ получить сломанную установку. Соглашайтесь (и помните: это разовая настройка, дальше агент работает с обычными правами).

PowerShell по умолчанию не даёт запускать скрипты — это его «политика выполнения». Официальный фикс: выполнить Set-ExecutionPolicy -ExecutionPolicy RemoteSigned и подтвердить. Команда разрешает запуск локальных скриптов и подписанных скриптов из интернета.

Не нужно. Codex работает на Windows нативно, и официальная рекомендация — использовать именно нативный режим. WSL (подсистема Linux) — опция для тех, кому нужны Linux-инструменты; новичку она только добавит сложности.

Памятка здесь называется AGENTS.md (глава 2.5). Сочетание Alt+V — фишка Claude Code; в приложении ChatGPT скриншот прикладывается кнопкой скрепки или перетаскиванием файла в поле ввода.

Команды /plugin — механика Claude Code, в Codex их нет. Прокачка Codex — скиллы (например, uipro init --ai codex) и правила в AGENTS.md.

На Free Codex формально доступен, но лимиты маленькие и заканчиваются быстро — это режим «попробовать». Если агент внезапно «устал» посреди работы — вы упёрлись в лимит плана, а не сломали его. Лимиты и планы: страница тарифов Codex.

2. Подготовь проект к публикации сам (настройки, сборка).
3. Опубликуй и дай мне ссылку. В конце объясни, как обновлять сайт после правок, — по шагам, на будущее.

Переписать тексты сайта под целевую аудиторию.

Показать текст промпта
Перепиши тексты моего сайта под целевую аудиторию, не меняя дизайн и структуру.

Аудитория: [КТО ЭТИ ЛЮДИ: возраст, чем занимаются, чего хотят].
Главное действие на сайте: [ЧТО ЧЕЛОВЕК ДОЛЖЕН СДЕЛАТЬ: оставить заявку / позвонить / купить].
Тон: [3 СЛОВА, например «дружелюбно, уверенно, без канцелярита»].

Работай в два этапа:
ЭТАП 1. Покажи новые тексты списком «было → станет» по каждому блоку сайта.
Файлы не трогай, жди «ок».
ЭТАП 2. После «ок» аккуратно замени тексты в файлах.

Правила:
- Факты о бизнесе (цифры, цены, отзывы) не выдумывай — помечай [В КВАДРАТНЫХ СКОБКАХ].
- Заголовки — короткие и конкретные, без «инновационных решений».

Справка

Глоссарий

Все термины гайда — простыми словами, по алфавиту.

Бэкенд
Невидимая часть: сервер, базы данных, обработка форм. Простому лендингу бэкенд часто вообще не нужен.
Деплой
Публикация проекта в интернете, чтобы он открывался по ссылке у всех, а не только у вас.
Домен
Адрес сайта, например site.ru. Покупается отдельно; для старта хватает бесплатного адреса от хостинга.
Итерация
Один круг «попросил → получил → посмотрел → уточнил». Хорошие продукты собираются десятком маленьких итераций, а не одной большой.
Команда
Строка текста, которую вы вводите в терминал, чтобы что-то произошло: установить программу, создать папку, запустить агента.
Коммит
Сохранённый «снимок» проекта в Git с подписью, что изменилось. Из коммитов складывается история.
Контекст
Всё, что агент «держит в голове» прямо сейчас: ваш разговор, открытые файлы, инструкции. Контекст не бесконечен — поэтому длинные задачи режут на шаги.
Папка проекта
Отдельная папка под одну идею — «рабочий стол» агента. Всё, что он создаёт и меняет, живёт внутри неё.
Плагин
Устанавливаемое расширение для агента: добавляет навыки, инструменты и правила. Как отправить сотрудника на курсы.
Промпт
Текстовое задание для нейросети или агента. Качество промпта напрямую определяет качество результата.
Репозиторий
Проект в Git/GitHub: папка с файлами плюс вся история их изменений.
Скилл
Один конкретный навык агента: папка с файлом-инструкцией, которую агент читает, когда навык становится нужен.
Статический сайт
Сайт из готовых файлов без сервера. Быстрый, дешёвый в размещении — идеальный первый продукт.
Терминал
Окно, где компьютером управляют текстовыми командами вместо кнопок. Выглядит сурово, работает просто: написал команду — нажал Enter.
Токен
Кусочек текста (примерно полслова), которыми нейросеть считает объём работы. Лимиты планов измеряются в токенах и сообщениях.
Фронтенд
Всё, что видно в браузере: вёрстка, тексты, кнопки, анимации.
Хостинг
Сервис, где живёт опубликованный сайт. GitHub Pages и Vercel — бесплатные варианты для статических сайтов.
AGENTS.md
То же самое для Codex: файл с правилами работы агента в этом проекте.
AI-агент
Программа-«сотрудник»: вы описываете задачу словами, а она сама пишет код, создаёт файлы и исправляет ошибки. Claude Code и Codex — агенты.
CLAUDE.md
Файл-«памятка» в папке проекта: Claude Code перечитывает его при каждом запуске и следует вашим правилам.
Git
Программа-«машина времени» для файлов проекта: запоминает версии, позволяет откатиться. Агенты используют её под капотом.
GitHub
Сайт, где хранят проекты (репозитории) и код. Там же живёт бесплатный хостинг GitHub Pages.
Localhost
Адрес «этот самый компьютер». Пока сайт не опубликован, вы смотрите его в браузере по адресу вроде localhost:3000.
PowerShell
Стандартный терминал Windows. Открывается через Win+X → «Терминал». Признак: строка начинается с PS C:\.

Для задач этого гайда — нет: код пишет агент, ваша работа — формулировать задачу и принимать решения. Понимать по чуть-чуть, что происходит, всё же полезно — для этого в гайде объясняется каждый термин, а внизу есть глоссарий.

Цены меняются, поэтому в гайде их нет — смотрите официальные страницы тарифов (ссылки в главах 1.1 и 2.1). Ориентир по фактам на дату проверки: Claude Code требует платный план (Pro и выше), а Codex формально доступен даже на бесплатном плане ChatGPT, но с маленькими лимитами — для реальной работы нужен Plus и выше.

Агент работает с файлами внутри папки проекта — за её пределы Codex не выходит без разрешения, а Claude Code спрашивает подтверждение на команды. Дополнительная страховка: правило «спрашивай перед удалением» в CLAUDE.md/AGENTS.md (шаблон в главах 1.5 и 2.5) и привычка не держать в папке проекта ничего постороннего.

Оба доведут вас до результата, конвейер гайда одинаковый для обоих. Практичный критерий: если вы уже платите за ChatGPT — начните с Codex; если нет — сравните таблицу выше и попробуйте путь Claude. Менять решение не страшно: система промптов переносится целиком.

Это нормальный режим работы, а не катастрофа. Опишите проблему по формуле из промпта «Почини»: что делали, что ожидали, что произошло, плюс скриншот. И просите правки по одной — маленькими итерациями, а не «переделай всё».

Файлы папки проекта читает агент, и они обрабатываются на серверах Anthropic или OpenAI по их политикам конфиденциальности. Практическое правило: не кладите в папку проекта пароли, сканы документов и чужие персональные данные — для сайта-продукта они и не нужны.
